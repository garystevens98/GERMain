HC HAvBED Main Screen

HC HAvBED Main Screen

Depending on the permissions assigned to the user, the HC HAvBED homepage will display the following, as shown in FIGURE 1-1:

  1. Request Updates - used to send a notification to users to update HAvBED data.
  2. Column Settings - select which columns to display.
  3. Expand - automatically adjusts column widths to fit the user's browser.
  4. Send to HHS - will send the current HAvBED data to HHS.

Main Screen

Requesting Updates

Requesting Updates

Users with the appropriate permissions can request facility updates, as shown in FIGURE 1-2:

  1. Select the Request Updates button.

Request Updates Button

Edit the request, as shown in FIGURE 1-3:

  1. Enter the Subject.
  2. Enter the Due Date.
  3. Enter a Message.
  4. Select Request to send the request.

OR

  1. Select Cancel to cancel the request.

Responding To A Request

Responding To A Request

Users on the receiving end of an update request will received a notification, as seen in FIGURE 1-4:

  1. Select the Input Required button.

Notification Bar

FIGURE 1-7 shows the input area for the notification:

  1. Enter data as necessary.
  2. If additional entities are available for the user to edit, use the arrows to cycle between entities.
  3. Select Dismiss to cancel changes.
  4. Select Submit to save changes.

Sending Data To HHS

Sending Data To HHS

Users with the appropriate permissions can send HAvBED data to HHS, as shown in FIGURE 1-4:

  1. Select the Send to HHS button.

Send to HHS

Follow the prompt, as shown in FIGURE 1-3:

  1. Select Transmit to send the data to HHS.

OR

  1. Select Cancel to cancel the transmission .

Viewing Submission History

Viewing Submission History

Users may view previously submitted HHS data transmissions, as shown in FIGURE 1-5:

  1. Select the History tab. Previous submissions and status will show.
  2. Select the View button on any submission to see the raw XML transmission that was sent to HHS.
