Archiving Patient Data

Archiving patient data is a way to remove patient information from the visible parts of the application, but doesn't actually delete any data. Archival of patient data should be carried out by an administrator that has appropriate access to the Patient Archive workspace.

To archive patient data, as shown in FIGURE 1-1:

  1. Select Workspaces.
  2. Select the Patient Archive workspace.

Workspaces

After the workspace has been selected, the user can navigate around the workspace, as shown in FIGURE 1-2

  1. Select the Patients to Archive matrix.

Select Matrix

 

Note: Only Administrators have access to the Patient Archive workspace.

The records that appear in this matrix are unarchived patients. The user may filter the matrix to display only certain patient records, as shown in FIGURE 1-3:

  1. Select the Sort and Filter button.

Sort and Filter button

When chosen, the Sort and Filter options appear over the open matrix view, as shown in FIGURE 1-4:

  1. Select the measure to filter from the Filters drop-down.
  2. Select the filter type from the drop-down.
  3. Enter the filter criteria that appears.
  4. Select the plus button.
  5. Select the APPLY button to confirm any changes made.

Sort and Filter options

Whether the user filters the matrix or not, the next step in archiving patient data is to select the desired entities to be archived, as shown in FIGURE 1-5:

  1. Select the circle to the left of the desired entities.
  2. Select the Bulk actions button.
  3. Select Edit.

Bulk Actions

Select the measures to update, as shown in FIGURE 1-6:

  1. Select the square for Incident.
  2. Select the square for Archive Event/Notes.
  3. Select Next to continue.

Bulk Edit - Select Measures

Enter the bulk edit data, as shown in FIGURE 1-7:

  1. Select Incident and select Archived Incident.
  2. Select Archive Event/Notes and enter the event name or other notes to identify the patient data.
  3. Select Submit when complete.

Bulk Edit - Update Measure Values

A save warning dialog will appear, as shown in FIGURE 1-8. The warning states that the user is trying to save data that will cause the records to disappear from the matrix view.  This is expected as the Patients to Archive matrix has a filter to only show patients that are not yet archived.

  1. Select OK to proceed.

Save Warning

Once the save warning is confirmed, the patients selected for archive will disappear from the Patients to Archive matrix. Administrators may review archived patient data by accessing the Archived Patients matrix in the Patient Archive workspace.
