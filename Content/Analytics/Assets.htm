Assets

Introduction

One of the major purposes of HC Analytics is to allow a user to keep track of important things. These things are called "Assets". An asset can be a building or a person or a thing.

The Assets app is used for cataloging assets. An asset is a physical object that has importance such as such as buildings or people. Depending on the type of asset, you may need to add different information. For example, a school would need information such as the grade level yet a shopping mall would not. Asset templates are a way to separate the different information needed. Selecting an asset template is the first step in creating a new asset. Once selected the user is able to build an overview of the asset. When the asset overview is saved, the user will see more fields to fill out to build up a knowledge base on that asset. The platform will continue to get new asset templates as time goes forward. Currently there are asset templates for places, and people.

Create An Asset

To create a new asset, as shown in FIGURE 1-4:

  1. Select the Create Asset [+] icon in the toolbar.

Create Asset Button

Select an asset template, as shown in FIGURE 1-4:

  1. Select the Template Type list and select an item.
  2. Select Save to continue.

Asset Template

Complete the asset creation, as shown in FIGURE 1-5:

  1. Enter a Name. This is the only required field.
  2. Enter additional fields as needed.
  3. Enter a Type, if desired. Type is used to further categorize an asset. For example, a hospital could be typed as either Public or Private.
  4. Enter tag and Select Add to add tags. Tags are keywords that can be searched and filtered on a map.
  5. Select Save to save the Asset.

The asset view page has widgets for Overview, Main Contact, Map, Details, Photos/Documents, Points of Contact, and Annotations, as shown in FIGURE 1-7:

Asset View

The Overview section will contain different information depending on the asset template. Fields that are shared amongst all asset templates are name, address, phone, owner, and type. The Main Contact will appear if contacts wereadded in the Points of Contact section. The Main Contact will contain information such as name, title, address, phone, and email. The Map section allows the user to add the location of the asset and have it displayed on a map. The Details section contains information regarding capacity, opened and closed dates, a description, email, fax, size, and website. In the Photos/ Documents section, a user can upload a document or photo where each upload is contained in its own widget. The Points of Contact section is used to add information for multiple contacts. Annotations is used to add comments.

Users with the appropriate permission may edit the information in each widget, as shown in FIGURE 1-8:
